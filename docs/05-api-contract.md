# 05 — API contract

The single source of truth for the wire between `tempo-sync` (Python) and `tempo-desktop` (TS).
Shapes derived directly from the existing `.jsx` files so the React side requires zero adaptation.

- Base URL: `http://127.0.0.1:8765`
- Auth: every request carries `X-Tempo-Token: <hex>`. The token is generated by the sidecar at first launch and written to stdout for the Tauri host to capture; it's the only thing it ever logs unredacted.
- Encoding: JSON. Time fields are ISO-8601 with timezone. Durations are seconds (integer or float).
- Units: server **always** returns SI metric. The desktop converts for `imperial` display. This makes API caching trivial.

> Code references below use `interface`/`type` for compactness — the actual response models are Pydantic on the wire, and `openapi-typescript` produces the TS types in `src/types.ts`.

---

## Common types

```ts
type Sport = "run" | "bike" | "swim" | "trail" | "lift" | "walk" | "hike" | "row" | "other";

interface Stat { label: string; value: string; unit: string; tone?: "lead"|"accent"|"dim"|null }
interface ZoneRow { z: 1|2|3|4|5; label: string; pct: number; min: number; max: number }
interface SyncStatus { lastSync: string|null; running: boolean; nextRun: string|null }
```

Numeric values are returned raw; **the server pre-formats display strings (`displayValue`) where the layout depends on text width** (paces, durations, scores). The hero stat strip and lap labels are good examples.

Feeds `Dashboard` and (via `state="empty"`) `DashboardEmpty`. Source files: `tempo/dashboard.jsx`, `tempo/dashboard-empty.jsx`.

interface DashboardResponse {

  bodyBattery: {

  } | null;

  sleep: {
    durationS: number;
    bedTimeLocal: string;             // "23:18"
    wakeTimeLocal: string;            // "07:00"
    stages: { deep: number; rem: number; light: number; awake: number };   // seconds
    hrvDelta: number | null;          // signed bpm
    score: number;
  } | null;

  lastSession: {
    id: number;
    sport: Sport;
    title: string;
    startTimeLocal: string;
    endTimeLocal: string;
    distanceKm: number;
    avgPaceDisplay: string;           // "4:03"
    avgPaceUnit: string;              // "/km" or "/100" or "kph"
    avgHr: number;
    tss: number;
    aerobicTE: number;
    chips: string[];                  // ["Tempo Z3-Z4","Gavre park","14°C · light wind"]
    routePolyline: string | null;
  } | null;

  plannedToday: {
    label: string;                    // "5×1k @ threshold"
    description: string;
    scheduledAtLocal: string | null;  // "18:00"
    structure: Array<{ kind: "wu"|"work"|"rest"|"cd"; durationS: number; zone: 1|2|3|4|5 }>;
    estDistanceM: number;
    estTss: number;
    estDurationS: number;
  } | null;

  week: Array<{
    dayLabel: string;                 // "Mon" .. "Sun"
    dateLocal: string;
    sport: Sport | null;
    isToday: boolean;
    label: string;                    // "Easy" "Tempo" "Rest"
    distanceKm: number;
    tss: number;
  }>;
  weekTotals: { volumeKm: number; tss: number; sessions: number; rampPct: number };
}

## `GET /activities/{id}`

Feeds `ActivityDetail` and its variants. Source: `tempo/activity.jsx`.

```ts
interface ActivityDetailResponse {
  id: number;
  sport: Sport;
  subSport: string | null;
  title: string;             // "Tempo Loop · 14.0 km"
  subtitle: string;          // "Run · Tuesday, May 12 · 06:42"
  accent: "run"|"bike"|"swim"|"trail"|"lift";   // drives the colour var on the page

  hero: Stat[];              // 8 entries matching heroDefault in activity.jsx

  map: {
    routeName: string;        // "ROUTE · GAVRE PARK LOOP · 14.02 KM"
    routeCoords: string;      // "47.218°N · -1.546°W · 06:42 → 07:39"
    polyline: string;         // encoded polyline
    fastestCallout: string;   // "3:50 /km"
    bbox: [number,number,number,number];
  } | null;

  // Synchronised 4-track chart — server downsamples to ~600 points
  streams: {
    n: number;                 // sample count
    timeS: number[];           // seconds from start
    hr: (number|null)[];
    pace: (number|null)[];     // s/km
    cadence: (number|null)[];
    elevation: (number|null)[];
    power?: (number|null)[];   // present if has_power
  };

  splits: Array<{
    k: string;                 // "1", "WU 1", "I 3", "R", "K 7"
    distanceM: number;
    pace: number;              // s/km
    paceDisplay: string;       // "3:50"
    hr: number;
    cad: number;
    elevDelta: number;         // m, signed
    zone: 1|2|3|4|5;
  }>;

  zones: ZoneRow[];            // 5 rows, sorted Z1→Z5

  hrSummary: { avg: number; max: number; hrrPct: number };

  dynamics: {                  // null if missing[].includes("dynamics")
    verticalOscCm: number;
    groundContactMs: number;
    strideLengthM: number;
    verticalRatioPct: number;
    leftPct: number;
    rightPct: number;
  } | null;

  trainingEffect: {
    headline: string;
    sub: string;
    aerobic: number; aeroLabel: string;
    anaerobic: number; anLabel: string;
    epoc: number;             // ml/kg
    load: number;             // tss
    intensity: number | null; // IF, null if no power
    recoveryHours: number;
  };

  missing: Array<"dynamics"|"power"|"map"|"weather">;
}
```

### Downsampling rules

- 4 tracks × ~600 points = 2.4k numbers per response. Compresses to ~12 kB. The whole detail payload fits under 40 kB.
- Algorithm: LTTB (`largest triangle three buckets`) per channel — preserves visual peaks the smoothing average would erase.
- If a stream is missing in source data, return `null` entries (do not omit the array — keeps lengths equal).

## Sync control

### `GET /sync/status`  (SSE stream)

```
event: status
data: {"running":true, "stage":"activities", "pulled":7, "total":12}

event: status
data: {"running":false, "lastSync":"2026-05-12T08:14:00Z", "nextRun":"2026-05-12T10:14:00Z"}

event: error
data: {"code":"rate_limited","retryAfterS":600}

event: mfa_required
data: {"hint":"Enter the 6-digit code from your authenticator app"}
```

### `POST /sync/run`

```json
{ "force": false, "scope": ["activities","health","load","progress"] }
```

Returns `202` with `{ "runId": 42 }`. Listen on `/sync/status` for progress.

### `POST /auth/login`

```json
{ "email": "you@…", "password": "…" }
```

`200` on success; `400 {"code":"mfa_required"}` triggers a second call:

```json
{ "email": "you@…", "password": "…", "mfa": "123456" }
```

### `POST /auth/logout`

Clears keychain. `204`.

### `GET /auth/status`

```json
{ "authenticated": true, "garminUserId": "abc123" }
```

## Errors

All non-2xx responses share this envelope:

```json
{ "code": "rate_limited", "message": "Garmin returned 429.", "retryAfterS": 600, "context": {} }
```

Codes the desktop handles explicitly: `unauthenticated`, `mfa_required`, `rate_limited`, `garmin_down`, `db_locked`, `not_found`.
